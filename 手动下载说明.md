# 手动下载 gradle-wrapper.jar 的步骤

## 🎯 必须完成的步骤

由于网络自动下载失败，请按照以下步骤手动获取 `gradle-wrapper.jar`：

---

## 📥 方法 1：下载并提取（推荐）

### 步骤 1：下载 Gradle
访问以下任一链接下载 Gradle 8.3：

- **官方链接（推荐）：** https://services.gradle.org/distributions/gradle-8.3-bin.zip
- **备用链接 1：** https://downloads.gradle.org/distributions/gradle-8.3-bin.zip
- **备用链接 2：** https://mirrors.cloud.tencent.com/gradle/gradle-8.3-bin.zip

### 步骤 2：解压文件
- 解压下载的 `gradle-8.3-bin.zip` 文件
- 找到解压后的 `gradle-8.3` 文件夹

### 步骤 3：复制 gradle-wrapper.jar
- 进入 `gradle-8.3\lib\` 目录
- 找到 `gradle-wrapper-8.3.jar` 文件
- 复制到项目目录：`c:\Users\jd\CodeBuddy\20260311175444\gradle\wrapper\`
- 重命名为 `gradle-wrapper.jar`

**完整路径：**
```
从：gradle-8.3\lib\gradle-wrapper-8.3.jar
到：c:\Users\jd\CodeBuddy\20260311175444\gradle\wrapper\gradle-wrapper.jar
```

### 步骤 4：验证
打开命令提示符，运行：
```bash
cd c:\Users\jd\CodeBuddy\20260311175444
dir gradle\wrapper\gradle-wrapper.jar
```

应该看到文件存在（约 60KB）

## ❓ 常见问题

### Q: 下载速度慢怎么办？
A: 使用腾讯云镜像：https://mirrors.cloud.tencent.com/gradle/

### Q: 下载的文件打不开怎么办？
A: JAR 文件不需要打开，只需要复制到指定位置

### Q: 文件重命名时出错怎么办？
A: 右键 → 重命名，或使用命令：
```bash
ren gradle-wrapper-8.3.jar gradle-wrapper.jar
```

### Q: 仍然无法构建怎么办？
A: 使用 GitHub Actions 构建（无需本地环境）
