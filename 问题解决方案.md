# 问题：无法加载 org.gradle.wrapper.GradleWrapperMain

## 错误信息
```
错误：无法找到或加载主类 org.gradle.wrapper.GradleWrapperMain
原因：java.lang.ClassNotFoundException：org.gradle.wrapper.GradleWrapperMain
```

## 根本原因
项目缺少 `gradle-wrapper.jar` 文件，这是 Gradle Wrapper 的核心组件。

---

## ✅ 解决方案（按推荐顺序）

### 方案一：通过 Git 推送到 GitHub，使用 GitHub Actions 构建（最可靠）

**步骤：**

1. **安装 Git**（如果尚未安装）
   - 下载：https://git-scm.com/download/win
   - 安装时选择默认选项

2. **初始化 Git 仓库并推送**
   ```bash
   cd c:\Users\jd\CodeBuddy\20260311175444
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/luogod770-ops/smart-elderly-app.git
   git push -u origin main
   ```

3. **创建 GitHub Actions 工作流**
   - 在仓库中创建文件：`.github/workflows/build.yml`
   - 内容如下：
   
   ```yaml
   name: Build APK
   
   on:
     push:
       branches: [ main ]
     workflow_dispatch:
   
   jobs:
     build:
       runs-on: ubuntu-latest
     
       steps:
       - name: Checkout code
         uses: actions/checkout@v4
       
       - name: Set up JDK 17
         uses: actions/setup-java@v4
         with:
           java-version: '17'
           distribution: 'temurin'
       
       - name: Grant execute permission for gradlew
         run: chmod +x gradlew
       
       - name: Build with Gradle
         run: ./gradlew assembleDebug
       
       - name: Upload APK
         uses: actions/upload-artifact@v4
         with:
           name: app-debug
           path: app/build/outputs/apk/debug/app-debug.apk
   ```

4. **触发构建**
   - 访问：https://github.com/luogod770-ops/smart-elderly-app/actions
   - 点击 "Run workflow" 开始构建
   - 等待构建完成（约5-10分钟）

5. **下载 APK**
   - 构建完成后，在 Actions 页面下载 artifact
   - 文件名：`app-debug.apk`

---

### 方案二：手动下载 gradle-wrapper.jar

**步骤：**

1. **下载 gradle-wrapper.jar**
   - 方法A：访问此链接直接下载
     https://github.com/gradle/gradle/raw/v8.3.0/gradle/wrapper/gradle-wrapper.jar
   
   - 方法B：使用备用源
     https://raw.githubusercontent.com/gradle/gradle/v8.3.0/gradle/wrapper/gradle-wrapper.jar

2. **保存文件**
   - 路径：`c:\Users\jd\CodeBuddy\20260311175444\gradle\wrapper\gradle-wrapper.jar`

3. **验证配置文件**
   - 确保 `gradle\wrapper\gradle-wrapper.properties` 存在
   - 内容应该是：
     ```
     distributionBase=GRADLE_USER_HOME
     distributionPath=wrapper/dists
     distributionUrl=https\://services.gradle.org/distributions/gradle-8.3-bin.zip
     networkTimeout=10000
     validateDistributionUrl=true
     zipStoreBase=GRADLE_USER_HOME
     zipStorePath=wrapper/dists
     ```

4. **运行构建**
   ```bash
   cd c:\Users\jd\CodeBuddy\20260311175444
   gradlew.bat assembleDebug
   ```

## 快速验证脚本

创建文件 `check_gradle.bat`：
```batch
@echo off
echo 检查 Gradle Wrapper 文件...
echo.

if exist gradlew.bat (
    echo [√] gradlew.bat 存在
) else (
    echo [×] gradlew.bat 不存在
)

if exist gradle\wrapper\gradle-wrapper.jar (
    echo [√] gradle-wrapper.jar 存在
) else (
    echo [×] gradle-wrapper.jar 不存在 - 需要下载！
)

if exist gradle\wrapper\gradle-wrapper.properties (
    echo [√] gradle-wrapper.properties 存在
) else (
    echo [×] gradle-wrapper.properties 不存在
)

if exist build.gradle.kts (
    echo [√] build.gradle.kts 存在
) else (
    echo [×] build.gradle.kts 不存在
)

echo.
pause
```

运行此脚本来检查项目完整性。
